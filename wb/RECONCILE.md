# Reconcile WB “Финансовые отчёты” (ЛК/выгрузка) vs наш API

## Источник истины

Для WB эталон берём из выгрузки **“Еженедельный детализированный отчёт … .csv”** (как в кабинете `Финансовые отчёты → Отчёты реализации`).

Скрипт считает метрики по тем же **колонкам**, что видны в кабинете/выгрузке:

- Продажи: `Вайлдберриз реализовал Товар (Пр)`
- К перечислению: `К перечислению Продавцу за реализованный Товар`
- Вознаграждение ВБ: `Вознаграждение Вайлдберриз (ВВ), без НДС` + `НДС с Вознаграждения Вайлдберриз`
- Эквайринг: `Эквайринг/Комиссии за организацию платежей`
- Доставка: `Услуги по доставке товара покупателю`
- Хранение: `Хранение`
- Возмещения: `Возмещение за выдачу и возврат товаров на ПВЗ` + `Возмещение издержек по перевозке/по складским операциям с товаром`
- Прочие/штрафы/корректировки: соответствующие колонки (если заполнены)

Важно: в WB часть колонок в файле уже со знаком (например, ВВ часто отрицательная), часть — как “стоимость” (положительное число, но по смыслу удержание). Скрипт приводит знаки к нашей семантике дерева:
- **начисления**: \(+\)
- **удержания**: \(-\)

## Где лежат файлы

Скрипт по умолчанию ищет CSV в папке `wb/` рядом с собой и берёт самые свежие по времени изменения:

- `wb/Еженедельный детализированный отчет*.csv`

Можно подать конкретные файлы флагами `--csv ...` (см. `--help`).

## Offline‑проверка (только CSV)

Из корня проекта:

```bash
python3 wb/reconcile_wb.py --date-from 2026-01-01 --date-to 2026-01-23
```

Вывод:
- итог по “Продажи/Выручка” (WB реализовал)
- итог “К перечислению”
- суммы по категориям (как в нашем WB дереве)

## Reconcile с нашим API `/dashboard/costs-tree` (нужен запущенный backend)

```bash
python3 wb/reconcile_wb.py \
  --reconcile-api \
  --date-from 2026-01-01 \
  --date-to 2026-01-23 \
  --api-base-url http://localhost:8000/api/v1
```

Печатает expected/got/diff по:
- top-level категориям
- `__total_accrued__` (к перечислению)

## JSON‑вывод (удобно в чат/issue)

```bash
python3 wb/reconcile_wb.py --date-from 2026-01-01 --date-to 2026-01-23 --json
```
