# Reconcile Ozon “Начислено за период” (ЛК) vs наш API

## Какие CSV нужны (и какие НЕ нужны)

Для мэтчинга главной карточки Ozon **“Начислено за период”** (как на скрине ЛК) используются **2 файла**:

1) **`Отчет по начислениям_*.csv`** — основной источник истины для виджета  
   Нужные колонки: `Группа услуг`, `Тип начисления`, `Сумма итого, руб.`

2) **`Отчет по товарам за период *.csv`** — второй независимый источник, должен давать тот же `Итого`  
   Нужные колонки: `Тип начисления`, `Итого, руб.`

Файл **`Юнит-экономика_*.csv`** для этой задачи **НЕ используется**: это другой отчёт с другой логикой отнесения.

## Где лежат файлы

Скрипт по умолчанию ищет CSV в папке `ozon/` рядом с собой:

- `ozon/Отчет по начислениям_*.csv`
- `ozon/Отчет по товарам за период *.csv`

Если ты скачал новые CSV и **заменил** старые — скрипт возьмёт “самые свежие” по времени изменения.

## Offline‑проверка (только CSV, без сервера)

Из корня проекта:

```bash
python3 ozon/reconcile_accruals.py
```

Что делает:
- считает итоги “как в ЛК” из `Отчет по начислениям`
- проверяет, что `Отчет по товарам` даёт тот же `Итого`
- печатает суммы по основным категориям

## Reconcile с нашим API `/dashboard/costs-tree` (нужен запущенный backend)

```bash
python3 ozon/reconcile_accruals.py \\
  --reconcile-api \\
  --date-from 2026-01-16 \\
  --date-to 2026-01-22 \\
  --api-base-url http://localhost:8000/api/v1
```

Вывод:
- по каждой категории (`Продажи`, `Вознаграждение Ozon`, `Услуги доставки`, …) покажет expected/got/diff
- отдельно покажет `API total_revenue`, если нужно разбирать расхождение процентов

## JSON‑вывод (удобно для вставки в issue/чат)

```bash
python3 ozon/reconcile_accruals.py --json
```

или с API:

```bash
python3 ozon/reconcile_accruals.py --reconcile-api --date-from 2026-01-16 --date-to 2026-01-22 --json
```
