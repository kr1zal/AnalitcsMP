### 1. Процент выкупа → Процент возвратов

**Проблема:**
- Текущая формула: `sales / orders * 100%` даёт 700%
- Заказы и продажи в разных временных периодах

**Решение:**
Изменить метрику на **"Процент возвратов" (Return Rate)**

**Формула:**
```
return_rate = (returns / (sales + returns)) * 100%
```

**Пример:**
- sales = 7, returns = 1
- return_rate = 1 / (7+1) * 100 = 12.5%

**Альтернативное название:** "Процент сохранности" = `(1 - return_rate)`

**Что показывать в дашборде:**
```
┌─────────────────────┬─────────────────────┬─────────────────────┬─────────────────────┐
│  Выручка            │  Чистые продажи     │  Процент возвратов  │  Средний чек        │
│  4,120 ₽            │  6 шт (7-1)         │  12.5%              │  687 ₽              │
│  ↑ +12%             │  ↑ +3 шт            │  ↓ -2% (хорошо!)    │  ↑ +5%              │
└─────────────────────┴─────────────────────┴─────────────────────┴─────────────────────┘
```

**Изменения в backend (опционально):**
- Добавить computed column `net_sales = sales_count - returns_count`
- Изменить формулу buyout_percent в sync_service.py

**Статус:** ⏳ Ждём подтверждения

### 3. Рекламные расходы (ACOS, ДРР)

**Решение:** Добавить базовую визуализацию с разбивкой по МП

**Что включить в MVP:**

#### 3.1 Карточка "Рекламные расходы"
```
┌─────────────────────────────┐
│ 📢 Рекламные расходы        │
│                             │
│ 1,250 ₽                     │
│ ACOS: 15.2% (норма <20%)    │
│                             │
│ WB: 850 ₽ | Ozon: 400 ₽     │
└─────────────────────────────┘
```

#### 3.2 Unit-Economics с рекламой
Добавить колонку "Реклама" в таблицу:

| Товар | Продажи | Выручка | МП расходы | **Реклама** | Чистая прибыль |
|-------|---------|---------|------------|-------------|----------------|
| Магний | 3 шт | 1,800 ₽ | 350 ₽ | **120 ₽** | 490 ₽ |

#### 3.3 Метрики рекламы
- **ACOS** (Advertising Cost of Sales) = Рекламные расходы / Выручка от рекламы * 100%
- **ДРР** (Доля Рекламных Расходов) = Рекламные расходы / Общая выручка * 100%
- **ROAS** (Return on Ad Spend) = Выручка / Рекламные расходы

#### 3.4 График эффективности рекламы
Отдельная страница или вкладка:
- Динамика ACOS по дням
- Сравнение WB vs Ozon
- Топ-3 самых эффективных кампаний

**Изменения в backend:**
- Добавить endpoint `GET /api/v1/dashboard/ad-performance`
- Возвращает агрегированные данные из `mp_ad_costs`

**Приоритет:** Средний (после основного дашборда)

**Статус:** ✅ Утверждено

### 6. Автообновление данных

**Решение:** По кнопке в MVP, автообновление после обкатки

**MVP (v1.0):**
- Кнопка "Обновить" в Header
- Manual refresh

**v1.1 (после тестирования):**
- React Query с `refetchInterval: 5 * 60 * 1000` (5 минут)
- Опция включить/выключить автообновление
- Индикатор последнего обновления ("обновлено 2 минуты назад")

**v2.0 (будущее):**
- WebSocket/SSE для real-time
- Push-уведомления о критических событиях

**Статус:** ✅ Утверждено
